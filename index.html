Learning Journey

Education & Learning Experience

My education and learning experience in programming has been a gradual journey from understanding basic ideas to building complete pages and solving practical problems. At the beginning, programming felt like a language with many small rules that had to be memorized. After more practice, I learned that the most important part is not only remembering syntax, but also understanding how to think step by step. In web programming, I started with the structure of an HTML document, including headings, paragraphs, lists, links, images, tables, forms, and semantic sections. These topics helped me understand that a webpage is not just visual decoration. It is a document with meaning, hierarchy, and purpose. When I write HTML correctly, the browser, the user, and assistive technologies can understand the content more clearly.

After HTML, I learned how CSS changes the appearance and layout of a page. CSS gave me the ability to control colors, spacing, typography, borders, backgrounds, grids, and responsive design. At first, I used CSS mainly to change colors and font sizes. Later, I began to see it as a design system that makes a website consistent and professional. I learned that good spacing can make text easier to read, that contrast is important for clarity, and that a page should work well on different screen sizes. This final exam website uses those lessons by separating structure from style, linking all pages to one CSS file, and using reusable classes for sections, buttons, cards, lists, and navigation.

JavaScript also became an important part of my learning because it showed me how websites can respond to user actions. Even simple scripts helped me understand events, variables, functions, conditions, loops, and document manipulation. When I practiced small exercises, I realized that every interactive page depends on clear logic. If the logic is confusing, the user experience becomes confusing too. This changed the way I approach coding. I now try to plan what the page should do before writing the code. I think about inputs, outputs, possible errors, and how the user will understand the result.

My experience in C programming supported my web programming learning as well. C taught me discipline because it requires careful attention to variables, data types, loops, arrays, functions, pointers, and memory. While web programming focuses strongly on presentation and user interaction, C programming trains the mind to solve problems with precise logic. The combination of both courses helped me become more confident. I can now look at a task, divide it into smaller parts, test each part, and improve the final result. I also learned that mistakes are a normal part of learning. Debugging is not failure; it is the process that turns an unclear idea into a working solution.

During this semester, I practiced by creating separate HTML files for different exercises. Each exercise taught me something specific, such as inserting images, building tables, creating forms, using links, adding multimedia, arranging content with CSS, and making pages more organized. These exercises were valuable because they turned theory into practice. Reading about a tag or style rule is useful, but using it in a real file makes the idea stronger. I also learned the importance of file names, folder organization, and relative paths. A website only works correctly when all files are connected in the right way.

Another important part of my learning experience is understanding how websites are published. GitHub Pages is a useful tool because it allows a student to make a project available online. Publishing a site teaches more than coding. It teaches responsibility, organization, version control, and testing. Before publishing, I need to check that the homepage is named index.html, that links are correct, that the CSS file loads, and that all pages open without errors. This final website represents that complete process. It is not only a collection of text, but a small portfolio that shows my progress, my course work, my C programming experience, and my reflection on what I learned. My goal is to continue improving by building more projects, reading documentation, solving exercises, and learning from each mistake until programming becomes a natural skill that I can use in academic and professional work.
